Лабораторна робота №1-2
Побудова та аналіз моделей ШІ з використанням TensorFlow і
PyTorch. Вивчення інструментів Mlflow, Optuna та Ray Tune
Мета роботи. Опанувати практику створення й оптимізації моделей
машинного навчання у TensorFlow та PyTorch, навчитися застосовувати різні
архітектури, регуляризацію та методи покращення узагальнюючої здатності
моделі, виконати порівняльний аналіз результатів.
Для лабораторної роботи №1 всі завдання потрібно виконати для
TensorFlow + Mlflow + Optuna, для лабораторної роботи №2 – PyTorch +
Mlflow + Ray Tune. Відповідно порівняння TensorFlow та PyTorch буде у
висновках лабораторної роботи №2
Завдання
1. Oзнaйoмитися з oснoвними тeoрeтичними вiдoмoстями, матеріалами
лекції тa рeкoмeндoвaнoю лiтeрaтурoю зa тeмoю рoбoти.
2. Обрати більш складний набір даних (Kaggle, UCI ML Repository),
наприклад, Credit Card Fraud Detection (класифікація з сильним дисбалансом) або
California Housing (регресія з великою кількістю ознак), або інший довільний
датасет.
3. Виконати розвідувальний аналіз даних (exploratory data analysis), зокрема
виконати візуалізацію розподілу ознак, здійснити кореляційний аналіз; виконати
виявлення дисбалансу класів.
4. Виконати попередню обробку даних, використати методи препроцесингу
SMOTE / undersampling / oversampling (для дисбалансу класів) або PCA або t-SNE
для зменшення розмірності (можна обрати інші довільні методи препроцесингу)
Вибір методу попередньої обробки залежить від обраного вами датасету та від
задачі, яку ви обрали (класифікація або регресія).
5. Виконати train/validation/test split.
6. Додати MLflow як обов’язкового шару відстеження експериментів (для
кожної моделі створити окремий run, логувати параметри, артефакти, найкращу
модель) (Локально без сервера).
7. Реалізувати дві архітектури нейронних мереж для обраного датасету
(для TensorFlow та PyTorch):
• базова feedforward мережа (мінімум 2 приховані шари);
• ускладнена мережа з використанням Dropout, Batch Normalization та L2-
регуляризації.
8. Провести експериментальне порівняння функцій активації ReLU,
LeakyReLU, GELU
Для кожної моделі потрібно використати мінімум дві різні активації та
зафіксувати результати.
9. Інтегрувати Optuna(лаб 1) / Ray Tune(лаб 2).
10. Виконати навчання протягом 10–20 епох з фіксованими
гіперпараметрами optimizer, learning rate.
11. Навести метрики оцінки якості роботи моделей (якщо був
дисбаланс класів, то використати відповідні метрики).
12. Побудувати навчальні криві та графіки втрат/метрик.
13. Порівняти TensorFlow та PyTorch, а також базову та оптимізовану
архітектура.
14. Виконати аналіз переваг та недоліків реалізацій у двох фреймворках,
визначити причин пере- чи недонавчання, проаналізувати які архітектури та
методи працюють краще для конкретного завдання. Виконати аналіз
використання інструментів Mlflow, Optuna та Ray Tune.
15. Оформити звіт. Структура звіту: титульний аркуш, мета роботи,
покроковий опис ходу роботи, приклад програмної реалізації, скріншоти з
демонстрацією результатів виконання програми, висновки.
